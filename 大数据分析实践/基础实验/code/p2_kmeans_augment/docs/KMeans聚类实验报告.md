## 2. 实验目标和实验环境

### 2.1 实验目标

本实验旨在利用 KMeans 无监督聚类算法，对 NBA 球员的技术统计特征进行聚类，发现球员的**技术风格自然分组**，并与真实的场上位置标签（Pos：C/PF/SF/SG/PG）进行对比验证，评估基于统计特征的聚类结果与人为位置定义之间的对齐程度。具体目标如下：

1. 采用与朴素贝叶斯实验一致的 per-36 预处理方案清洗数据；
2. 使用轮廓系数自动选择最优 K 值，同时运行 K=5 作为对照方案；
3. 通过 PCA 降维可视化和交叉表分析，探索聚类结果与真实位置的关系；
4. 分析 KMeans 在 NBA 统计数据上聚类效果的评价指标，理解无监督聚类在该任务中的局限性。

## 3. 实验内容

### 3.1 算法原理

KMeans 是一种基于距离的迭代式聚类算法。给定数据集 $X = \{x_1, x_2, ..., x_n\}$ 和预定的簇数 $K$，算法将数据划分为 $K$ 个簇 $\{C_1, C_2, ..., C_K\}$，使得**簇内平方和（Inertia）**最小化：

$$\text{Inertia} = \sum_{k=1}^{K} \sum_{x_i \in C_k} \|x_i - \mu_k\|^2$$

其中 $\mu_k$ 是簇 $C_k$ 的质心（所有样本点的均值）。算法通过交替执行以下两步实现迭代优化：

1. **分配步骤**：将每个样本点分配到距离最近的质心所在的簇；
2. **更新步骤**：重新计算每个簇的质心（该簇所有样本的均值）。

算法重复以上两步直到质心不再变化或达到最大迭代次数。

### 3.2 K 值选择策略

由于 KMeans 需预先指定 K 值，本实验采用双重策略：

- **轮廓系数（Silhouette Score）**：在 K=2~10 范围内扫描，选取轮廓系数最高的 K 值。轮廓系数综合衡量簇内紧密度和簇间分离度，取值范围 $[-1, 1]$，越高表示聚类结构越清晰；
- **设计参考 K=5**：同时运行 K=5 聚类，便于与 5 个场上位置进行对比验证。

$$\text{Silhouette}(i) = \frac{b(i) - a(i)}{\max\{a(i), b(i)\}}$$

其中 $a(i)$ 是样本 $i$ 与同簇其他样本的平均距离，$b(i)$ 是样本 $i$ 与最近邻簇中所有样本的平均距离。

### 3.3 特征选择

本实验选取 **9 维 per-36 特征**，聚焦出手选择而非命中率：

| 特征 | 含义 | 选取理由 |
|------|------|---------|
| PTS_per36 | 每36分钟得分 | 得分能力，per-36消除上场时间差异 |
| TRB_per36 | 每36分钟篮板 | 内线球员篮板显著高于后卫 |
| AST_per36 | 每36分钟助攻 | 控卫助攻显著高于其他位置 |
| STL_per36 | 每36分钟抢断 | 外线防守能力 |
| BLK_per36 | 每36分钟盖帽 | 内线防守能力 |
| TOV_per36 | 每36分钟失误 | 持球多的球员失误更高 |
| PF_per36 | 每36分钟犯规 | 内线球员犯规偏多 |
| **3PAr** | 三分出手占比 = 3PA / FGA | 反映出手选择倾向，替代 3P% |
| **FTr** | 罚球出手比 = FTA / FGA | 反映造罚球侵略性，替代 FT% |

> 相比于原始的场均特征（PPG/APG/RPG 等），per-36 化消除了上场时间带来的伪方差；用 3PAr 和 FTr 替代命中率，聚焦于球员的**出手选择偏好**而非投射准度。

## 4. 实验步骤和结果

### 4.1 数据预处理

预处理方案与朴素贝叶斯实验保持一致，包括 per-36 衍生、TOT 去重和行过滤：

```python
# Per-36 衍生
per36_map = {
    'PTS_per36': 'PTS', 'TRB_per36': 'TRB', 'AST_per36': 'AST',
    'STL_per36': 'STL', 'BLK_per36': 'BLK',
    'TOV_per36': 'TOV', 'PF_per36': 'PF',
}
for new_f, raw_f in per36_map.items():
    df[new_f] = df[raw_f] / df['MP'].replace(0, np.nan) * 36

# 3PAr 与 FTr
df['3PAr'] = df['3PA'] / df['FGA'].replace(0, np.nan)
df['FTr'] = df['FTA'] / df['FGA'].replace(0, np.nan)

# TOT 去重：每个 (Year, Player) 组若有 TOT 行，只保留 TOT 行
has_tot = df.groupby(['Year', 'Player'])['Tm'].transform(lambda x: (x == 'TOT').any())
df = df[~(has_tot & (df['Tm'] != 'TOT'))].copy()

# 行过滤
df = df[df['MP'] >= 200]
df = df[df['G'] >= 10]
```

> **代码分析**：Per-36 归一化将所有球员的统计值标准化到相同的上场时间基准，消除了"上场机会"和"单位效率"的混叠。TOT 去重确保每个球员赛季只出现一次。MP>=200 保证 per-36 指标在小样本上的可靠性。

预处理结果：

| 步骤 | 记录数 | 变化 |
|------|--------|------|
| 原始加载 | 18,727 | — |
| TOT 去重 + 过滤 | **13,273** | 4,454 条被剔除 |

各位置分布均匀：C=2,641, PF=2,770, SF=2,612, SG=2,641, PG=2,609。

### 4.2 特征标准化

KMeans 基于欧氏距离计算相似度，不同特征的量纲差异会直接扭曲距离计算。例如 PTS_per36 的量级约为 14，而 3PAr 的量级约为 0.16，若不标准化，PTS_per36 会主导距离计算。

```python
scaler = StandardScaler()
X_scaled = scaler.fit_transform(X)
```

> **代码分析**：`StandardScaler` 对每个特征进行 Z-score 标准化，使其均值为 0、标准差为 1。这样每个特征在距离计算中的贡献权重相同。这是 KMeans 预处理的关键步骤，与 Naive Bayes 不同——Naive Bayes 基于概率建模，不需要标准化。

| AST_per36 | 3.182 | 2.140 |
| STL_per36 | 1.182 | 0.497 |
| BLK_per36 | 0.762 | 0.767 |
| TOV_per36 | 2.208 | 0.727 |
| PF_per36 | 3.669 | 1.304 |
| 3PAr | 0.158 | 0.181 |
| FTr | 0.312 | 0.142 |

### 4.3 K 值选择

在 K=2~10 范围内扫描，记录每个 K 值的 Inertia 和 Silhouette Score：

| K | Inertia | Silhouette |
|---|---------|------------|
| 2 | 87,895 | **0.2447** |
| 3 | 75,184 | 0.2031 |
| 4 | 68,609 | 0.1809 |
| 5 | 64,132 | 0.1645 |
| 6 | 60,653 | 0.1502 |
| 7 | 57,902 | 0.1506 |
| 8 | 55,438 | 0.1437 |
| 9 | 53,408 | 0.1421 |
| 10 | 51,641 | 0.1429 |

轮廓系数在 K=2 时最高（0.2447），随后单调递减并趋于平缓。Inertia 随 K 增加持续下降，未出现明显的"肘点"。

```python
# K 值扫描核心代码
for k in range(2, 11):
    km = KMeans(n_clusters=k, random_state=42, n_init=20)
    labels = km.fit_predict(X_scaled)
    inertias.append(km.inertia_)
    sil = silhouette_score(X_scaled, labels)
    sil_scores.append(sil)
```

> **代码分析**：`random_state=42` 保证结果可复现。`n_init=20` 表示算法用 20 组不同的初始质心运行并选取最优结果，避免陷入局部最优。但 KMeans 对初始质心敏感的特性在本文 NBA 数据集上影响不大，因为数据点分布连续、无明显间隙。

**结论**：最优 K=2，但 K=2 将球员简单分为"内线"和"外线"两个超大类，不足以反映 5 个位置的角色差异。同时运行 K=5（与 5 个位置对应）作为对照方案。

### 4.6 评估指标与结果分析

#### 4.6.1 内部评估（Silhouette Score）

| 方案 | Silhouette |
|------|-----------|
| K=2 | 0.2447 |
| K=5 | 0.1645 |

K=5 的轮廓系数仅 0.1645，属于**弱聚类结构**（一般认为 >0.5 为合理，>0.7 为良好）。这与以下观测一致：

- PCA 二维投影中各类之间无清晰间隙，数据点呈连续分布；
- 簇 0 和簇 4 都是 C/PF 占主导的内线簇，但被 KMeans 强行拆分为"传统护框中锋"（簇 4）和"蓝领内线"（簇 0），两者之间没有明显分界；
- 簇 1 和簇 3 都是 SF/SG 为主的侧翼簇，区别仅在于簇 1 得分更高、簇 3 三分出手更多——反映了侧翼球员内部的渐变。

#### 4.6.2 外部评估（ARI / NMI）

| 方案 | ARI | NMI |
|------|-----|-----|
| K=2 | 0.2585 | 0.3647 |
| K=5 | 0.2297 | 0.3074 |

ARI 和 NMI 均为较低水平（ARI 接近 0 表示与随机划分无异）。K=5 的 ARI=0.2297 意味着聚类结果仅比随机划分好约 23%。具体原因如下：

**原因一：篮球位置是连续谱，不是离散簇。**

NBA 的 5 个位置之间存在大量"tweener"球员——组织前锋（如 LeBron James 打 SF 却持球组织）、空间型四号位（如 Kevin Love 打 PF 却大量投三分）、双能卫（如 James Harden 打 SG 却担任主控）。球员的技术特征是渐变的，不是泾渭分明的簇状分布。轮廓系数的本质是衡量"类内紧凑度 vs 类间分离度"，在连续渐变的数据中，类与类之间没有明显"间隙"，轮廓系数自然偏低。

**原因二：部分位置的特征高度重叠。**

K=5 交叉表清晰地显示了重叠模式：
- SF（小前锋）是最大的"摇摆者"——846 人落入簇 1（得分侧翼），1,000 人落入簇 3（空间侧翼），568 人落入簇 0（内线），均匀分布在 3 个不同的簇中，说明 SF 缺乏独特的统计特征；
- PF 同样分散——1,278 人在簇 0（蓝领内线），625 人在簇 1（得分侧翼），说明 PF 正处于从传统内线向空间型前锋演化的过程中；
- PG 则高度集中在簇 2（73.5%），助攻特征是 PG 最具区分度的信号。

**原因三："用量"维度的方差淹没了"位置风格"的方差。**

从簇中心可以看到一个规律：簇 1（PTS_per36=19.4）和簇 2（PTS_per36=13.9）和簇 0（11.7）——PTS 沿簇明显变化。KMeans 优先捕捉的是"得分产出"的差异，而不是"内线 vs 外线"的风格差异。这是因为得分相关的特征方差（PTS_per36 标准差 4.32）远大于位置风格的方差（如 3PAr 标准差 0.18）。KMeans 基于欧氏距离，高方差特征自然主导聚类结果。

**原因四：PCA 二维投影信息损失较大。**

前两个主成分仅能解释 56.0% 的方差，说明 9 维数据在二维平面上投影时信息损失严重。这意味着 PCA 散点图中看似重叠的簇，在高维空间中可能包含更多可分信息。但 silhouette 是基于 9 维全空间计算的（0.1645），即便在高维空间中，聚类结构仍然不强。

**原因五：无监督聚类与人为标签的天然偏差。**

ARI/NMI 衡量的是聚类结果与真实位置标签的一致性。但"位置"（Pos）是教练根据球员身高、防守对位、战术角色等因素人为分配的场上标签，并不完全由技术统计决定。例如一个场均 15 分 5 篮板 5 助攻的球员可能是 PG、SG 或 SF，取决于球队的阵容配置。用纯统计特征去还原位置标签，本质上是用"数据"去拟合一个受非统计因素影响的分类，ARI 偏低是预期之内的事。

### 4.7 PCA 散点图分析

PCA 散点图（聚类着色 vs 真实位置着色）揭示了以下结构：

- 沿 PC1 方向，从左到右大致是"外线 → 内线"的渐变过程；
- 真实位置着色图中，C 和 PG 分别位于两端的核心区域，而 PF、SF、SG 分布在中间过渡带；
- K=2 的聚类边界大致沿着 PC1 的中线切割，将 C/PF 与 PG/SG/SF 分开；
- K=5 的聚类则在此基础上进一步细分，但由于数据连续，簇边界处存在大量重叠。

## 5. 结论与讨论

### 5.1 实验结论

1. **KMeans 在 NBA 球员数据上的聚类效果有限**。K=5 时轮廓系数仅 0.1645，ARI=0.2297，表明基于 per-36 技术统计特征的聚类结果与真实位置标签的对齐程度较弱。

2. **最优 K=2 反映数据的二分结构**。K=2 将球员分为"内线"（高篮板、高盖帽、不投三分）和"外线"（高助攻、高抢断、大量三分）两个超大类，C 的 97.8% 和 PG 的 99.8% 被正确归入对应簇。但当 K 增加到 5 时，内线/外线内部的细分簇之间没有明显分界。

3. **部分位置在特征空间中高度重叠**。C 和 PG 是两个最极端的位置，分别以篮板/盖帽和助攻为独有标志；SF 是最大的"摇摆者"，均匀分布在多个簇之间；PF 和 SG 也有大量球员跨界分布。

4. **per-36 归一化消除了上场时间差异**，但未能改变 KMeans 在连续分布数据上的根本局限——当数据是渐变而非分簇时，基于距离的聚类方法天然难以产生高轮廓系数的结果。

### 5.2 KMeans 低分原因总结

| 原因 | 影响 | 说明 |
|------|------|------|
| 位置本质是连续谱 | 轮廓系数低 | NBA 位置存在大量"tweener"，数据渐变而非分簇 |
| 用量方差 > 风格方差 | 聚类方向偏离 | KMeans 优先按"得分产出"分裂，而非按位置风格 |
| PCA 信息损失 | 可视化误判 | 前两维仅解释 56% 方差，高维重叠被低估 |
| 位置标签的非统计性 | ARI/NMI 天然偏低 | 位置受身高、战术、防守对位等非统计因素影响 |

### 5.3 朴素贝叶斯与 KMeans 的对照

在相同预处理和特征集下，朴素贝叶斯的 5 分类 Accuracy=0.6138（远超随机 20%），而 KMeans 的 ARI=0.2297（仅比随机好 23%）。这一差距源于两类问题的本质不同：

- **朴素贝叶斯是监督学习**：知道位置标签，学习每个位置的条件概率分布。当位置之间有足够区分性时（如 C 的高篮板、PG 的高助攻），NB 可以有效利用；
- **KMeans 是无监督学习**：仅靠特征间的距离组织数据。当数据分布连续时，没有"正确"的分簇方式。K=5 的分簇与 5 个位置之间的对应仅 23% 优于随机。

### 5.4 讨论

1. **per-36 化对 KMeans 的影响**：在原始实验中（per-game 特征），KMeans 的聚类直接按"用量"（PPG 从 4→11→17）分裂，替补/首发被分到不同簇。引入 per-36 化后，这一现象有所缓解——簇 0 和簇 4 的 PTS_per36 分别为 11.7 和 12.3，差距大幅缩小。然而，簇 1（PTS_per36=19.4）依然是高得分簇，说明即使在单位时间效率层面，"得分产出"仍然是最主要的方差来源。

2. **3PAr 和 FTr 的有效性**：这两个出手选择比率的引入增强了区分度。簇 3 以高 3PAr(0.383) 为特征，有效识别了"空间型侧翼"这一技术风格。簇 4 的高 FTr(0.477) 和极低 3PAr(0.010) 则精准刻画了"传统内线"的特征。在原始实验基于命中率的特征集中，这些风格差异难以被捕捉。

3. **聚类结果的现实意义**：虽然 KMeans 的聚类与位置标签对齐有限，但聚类结果本身揭示了 NBA 技术风格的**自然分组结构**——内线护框者（簇 4）、蓝领内线（簇 0）、得分侧翼（簇 1）、空间侧翼（簇 3）、控球后卫（簇 2）。这种基于数据驱动的技术风格划分，可能比人为定义的 5 个位置更能反映球员的实际打法。

4. **未来改进方向**：
   - 使用高斯混合模型（GMM）替代 KMeans，允许簇具有不同形状和大小，更适合连续分布数据；
   - 加入球员身体测量数据（身高、臂展、体重），这些物理特征对位置判定有直接贡献；
   - 尝试谱聚类或 DBSCAN，前者能发现非球形簇，后者无需预设 K 值；
   - 降维方法（t-SNE 或 UMAP）替代 PCA 进行可视化，可能揭示更清晰的数据结构。

### 5.5 实验心得

通过本次实验，深入理解了 KMeans 聚类算法的特性和局限性。KMeans 假设数据呈球形簇分布，而 NBA 球员数据是连续渐变的谱系——从纯内线到纯外线有无数中间态。这一根本矛盾决定了聚类效果的上限。

与朴素贝叶斯实验对照，更清晰地认识到了监督学习与无监督学习的根本差异：NB 用位置标签指导分类边界的学习，KMeans 仅靠特征间的距离自主组织。在 NBA 位置预测这种"类间模糊"的任务中，两者的性能差距正是"有监督"信息价值的体现。

此外，特征工程（per-36、3PAr/FTr）虽然改善了聚类结果的可解释性（簇的技术画像更加清晰），但无法改变数据连续分布的结构性事实。这提醒我们在解决实际问题时，应充分考虑算法假设与数据特性是否匹配：球员工艺的技术风格本质上是一个连续谱，用聚类算法寻找离散分组本身就面临着方法论层面的挑战。
