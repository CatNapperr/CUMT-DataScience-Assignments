## 2. 实验目标和实验环境

### 2.1 实验目标

本实验旨在利用朴素贝叶斯（Naive Bayes）分类算法，根据 NBA 球员的历史技术统计特征，预测其场上位置。实验设两个子任务：

1. **5 位置细粒度分类**：预测球员的具体位置（C 中锋、PF 大前锋、SF 小前锋、SG 得分后卫、PG 控球后卫）；
2. **3 超类粗粒度分类**：将混淆严重的相邻位置合并为 Big（C+PF）、Wing（SF+SG）、PG 三个超类，验证标签粒度对分类性能的影响。

预处理阶段引入 per-36 分钟归一化和 TOT 去重等改进，以消除上场时间和数据冗余带来的噪声。

## 3. 实验内容

### 3.1 算法原理

朴素贝叶斯是一种基于**贝叶斯定理**的概率分类器，其核心假设是特征之间**条件独立**（朴素假设）。对于给定的输入样本 $x = (x_1, x_2, ..., x_n)$，算法计算其属于类别 $C_k$ 的后验概率：

$$P(C_k | x) = \frac{P(C_k) \prod_{i=1}^{n} P(x_i | C_k)}{P(x)}$$

并选择后验概率最大的类别作为预测结果。

本实验采用 **GaussianNB**，它假设每个类别中连续特征的似然 $P(x_i | C_k)$ 服从高斯分布：

$$P(x_i | C_k) = \frac{1}{\sqrt{2\pi\sigma_{k,i}^2}} \exp\left(-\frac{(x_i - \mu_{k,i})^2}{2\sigma_{k,i}^2}\right)$$

其中 $\mu_{k,i}$ 和 $\sigma_{k,i}^2$ 分别是类别 $C_k$ 中第 $i$ 个特征的均值和方差，由训练数据估计得到。

### 3.2 特征选择

本实验选取 **12 维特征**，相比原实验的 8 维场均特征，进行了以下改进：
- **per-36 分钟归一化**替代场均化，消除上场时间差异；
- **eFG%** 替代 FG%，引入三分加权；
- 新增**出手密度特征**（3PA_per36、FTA_per36）和**效率特征**（TOV_per36、PF_per36）。

| 编号 | 特征 | 含义 | 选取理由 |
|------|------|------|---------|
| 1 | PTS_per36 | 每36分钟得分 | 得分能力，per-36 消除上场时间差异 |
| 2 | TRB_per36 | 每36分钟篮板 | 中锋/大前锋篮板显著高于后卫 |
| 3 | AST_per36 | 每36分钟助攻 | 控球后卫助攻显著高于其他位置 |
| 4 | STL_per36 | 每36分钟抢断 | 外线球员抢断更多 |
| 5 | BLK_per36 | 每36分钟盖帽 | 中锋盖帽远高于其他位置 |
| 6 | 3P% | 三分命中率 | 外线球员三分更好 |
| 7 | FT% | 罚球命中率 | 后卫罚球更准 |
| 8 | eFG% | 有效命中率 | 含三分加权，比 FG% 更反映真实投篮效率 |
| 9 | 3PA_per36 | 每36分钟三分出手 | 三分出手倾向，区分"不投"和"投不进" |
| 10 | FTA_per36 | 每36分钟罚球出手 | 造罚球倾向，内线球员通常更高 |
| 11 | TOV_per36 | 每36分钟失误 | 持球多的球员失误更高 |
| 12 | PF_per36 | 每36分钟犯规 | 内线球员犯规偏多 |

### 3.3 对照实验设计

| 实验组 | 特征维度 | 特征集 | 目的 |
|--------|---------|--------|------|
| 5 位置分类 | 12 维 | 上述 12 维 | 基线细粒度分类 |

## 4. 实验步骤和结果

### 4.1 数据预处理

#### 4.1.1 衍生 per-36 指标

原始数据中所有技术统计为赛季累计值（PTS、TRB、AST 等），直接使用会同时受出场次数 G 和上场时间 MP 的影响。本实验采用 **per-36 分钟归一化**替代原先的场均化：

```python
# Per-36 衍生字段核心代码
per36_map = {
    'PTS_per36': 'PTS', 'TRB_per36': 'TRB', 'AST_per36': 'AST',
    'STL_per36': 'STL', 'BLK_per36': 'BLK', 'PF_per36': 'PF',
    'TOV_per36': 'TOV', '3PA_per36': '3PA', 'FTA_per36': 'FTA',
}
for new_f, raw_f in per36_map.items():
    df[new_f] = df[raw_f] / df['MP'].replace(0, np.nan) * 36
```

> **代码分析**：为什么 per-36 比场均更好？场均数据仍留有噪声——一个场均 20 分的球员可能是场均 35 分钟的得分手，也可能只是场均 25 分钟的高效替补。per-36 将所有球员归一化到同等上场时间，分离了"上场机会"和"单位效率"，使分类器学到更纯粹的位置技能信号。实际上，per-36 化后 PTS 与 TRB 的相关系数从原始场均数据的 **0.75 骤降至 -0.06**，证明原来的强相关完全是上场时间的伪相关。

#### 4.1.2 eFG% 补充

```python
df['eFG%'] = df['eFG%'].fillna((df['FG'] + 0.5 * df['3P']) / df['FGA'])
```

eFG%（有效命中率）给三分球赋予 1.5 倍权重，更准确反映真实投篮效率。数据集中部分 eFG% 已存在，未计算的行在此补充。

#### 4.1.3 缺失值填充

```python
for col in ['FG%', '3P%', 'FT%', '2P%', 'eFG%']:
    df[col] = df[col].fillna(0)
```

缺失情况：3P% 缺失最多（3,485 条），集中在三分未普及的早期赛季。所有缺失值均填充为 0，因为缺失原因均为对应出手数为 0。

#### 4.1.4 TOT 去重（关键改进）

这是与原始实验最核心的预处理改进。原始数据中，单赛季更换球队的球员同时存在 TOT 合计行和多个单球队行：

```
1980,Tim Bassett,PF,28,TOT,12,164,...  ← 合计行（12 场）
1980,Tim Bassett,PF,28,NJN,7,92,...   ← 仅在 NJN（7 场）
1980,Tim Bassett,PF,28,SAS,5,72,...   ← 仅在 SAS（5 场）
```

若直接剔除 TOT 而保留单球队行，同一个球员同一年会多次出现，随机划分时可能导致**数据泄露**（同一人的不同分片同时进入训练集和测试集）。

```python
# TOT 去重核心代码
# 每个 (Year, Player) 组若有 TOT 行，只保留 TOT，删单球队行
has_tot = df.groupby(['Year', 'Player'])['Tm'].transform(
    lambda x: (x == 'TOT').any()
)
df = df[~(has_tot & (df['Tm'] != 'TOT'))].copy()
```

> **代码分析**：`groupby(['Year', 'Player'])` 按球员-赛季分组。`transform` 给组内每行打标——若该组有 TOT 行，则整组标记为 True。逻辑取反 `~(has_tot & Tm!='TOT')` 的含义是：有 TOT 的组中只保留 TOT 行，无 TOT 的组全部保留。这样每条记录 = 一个球员一个赛季，没有重复，保证训练集和测试集之间无信息泄露。

#### 4.1.5 行过滤

```python
df = df[df['MP'] >= 200]   # 上场时间足够长，保证 per-36 可靠
df = df[df['G'] >= 10]     # 出场次数足够多
```

相比原始实验的 `G >= 20`，改用 **MP >= 200** 更为精确。一个 G=82 场均 2.5 分钟的球员 MP≈205，其 per-36 数据虽然通胀但样本量尚可；而 G=20 场均 5 分钟的球员 MP≈100，per-36 的可靠性较差。MP>=200 是更合理的底线。

#### 4.1.6 预处理结果汇总

|------|--------|------|

| TOT 去重 | 15,760 | 剔除 2,967 条单球队行 |
| MP>=200, G>=10 过滤 | **13,273** | 剔除 2,487 条低上场时间记录 |

### 4.2 标签编码与数据集划分

#### 4.2.1 标签编码

```python
# 5 分类标签编码
le5 = LabelEncoder()
le5.fit(['C', 'PF', 'SF', 'SG', 'PG'])
y5 = le5.transform(df['Pos'])

# 3 超类映射
POS_TO_SUPER = {'C': 'Big', 'PF': 'Big', 'PG': 'PG', 'SF': 'Wing', 'SG': 'Wing'}
df['SuperPos'] = df['Pos'].map(POS_TO_SUPER)
le3 = LabelEncoder()
le3.fit(['Big', 'Wing', 'PG'])

```

> **代码分析**：使用 `LabelEncoder` 将字符串位置标签映射为整数编码。5 分类显式固定顺序 C=0, PF=1, SF=2, SG=3, PG=4，避免编码混乱。3 超类将 C+PF 归为 Big（内线）、SF+SG 归为 Wing（侧翼）、PG 单独保留。

### 4.3 模型训练

#### 4.3.1 训练函数

```python
def train_and_eval(X_tr, X_te, y_tr, y_te, name, label_names):
    model = GaussianNB()
    model.fit(X_tr, y_tr)
    y_pred = model.predict(X_te)
    y_proba = model.predict_proba(X_te)

    acc = accuracy_score(y_te, y_pred)
    report = classification_report(y_te, y_pred, target_names=label_names,
                                   output_dict=True, zero_division=0)
    cm = confusion_matrix(y_te, y_pred)
    return {'model': model, 'y_pred': y_pred, 'y_proba': y_proba,
            'accuracy': acc, 'report': report, 'confusion_matrix': cm}
```

> **代码分析**：`GaussianNB()` 无需调参即可直接训练，因为高斯朴素贝叶斯没有超参数需要调整。`predict_proba()` 输出每个样本属于各类别的概率，可用于分析分类置信度。`zero_division=0` 防止在罕见类别的评估中除零报错。

#### 4.3.2 两组实验训练

```python
# 5 分类
res5 = train_and_eval(X_train, X_test, y5_train, y5_test,
                      "5 分类 (12 维)", POS_ORDER_5)
```

**5 分类混淆矩阵（百分比）：**

**关键混淆规律：**

3. **SG ↔ PG 后场混淆**：25.8% 的 SG 被误判为 PG，29.4% 的 PG 被误判为 SG，双能卫趋势在现代篮球中持续增强；
4. **SF 的识别率大幅提升**（83.3% vs 原始 31.2%），per-36 有效分离了侧翼和内线的信号。

## 5. 结论与讨论

### 5.1 实验结论

1. **per-36 分钟归一化显著提升了朴素贝叶斯的分类性能**。5 分类 Accuracy 从 0.537 提升至 **0.614**（+7.6 个百分点），3 超类 Accuracy 从 0.758 提升至 **0.827**（+6.8 个百分点）。其中 SF 的 Recall 从 31.2% 飙升至 **83.3%**，说明 per-36 化有效分离了侧翼与其他位置在单位时间效率上的差异。

2. **5 分类任务中各位置可分性差异显著**：SF（F1=0.83）和 C（F1=0.69）最易识别，PF（F1=0.45）和 SG（F1=0.48）最易混淆。内线混淆（C↔PF）和后场混淆（SG↔PG）是分类性能的主要瓶颈。

3. **3 超类分类效果良好**（Accuracy=0.827, Macro F1=0.827），Big 和 Wing 的识别率均达到 87%。5→3 映射与直接 3 超类性能几乎一致，说明 GaussianNB 的决策边界天然形成 3 个超类簇。

4. **TOT 去重消除数据泄露风险**。原始实验直接剔除 TOT 行保留了单球队行，导致同一球员赛季多次出现。本实验的 TOT 去重策略保证了训练集和测试集之间无球员赛季级别的信息泄露，评估指标更加可信。

5. **特征共线性控制良好**。12 维特征中仅有 AST_per36↔AST_TOV（r=0.804）一对强相关，其余特征间相关性均低于 0.7，符合朴素贝叶斯条件独立假设的容忍范围。

### 5.2 与原始实验的对比分析

本实验相对于原始实验的主要改进点及效果总结如下：

| 改进项 | 原始做法 | 本实验做法 | 效果 |
|--------|---------|-----------|------|
| 数据归一化 | 场均化（per-game） | **per-36 分钟** | SF Recall 31%→83% |
| TOT 处理 | 直接剔除 | **保留合计行，删单球队行** | 消除数据泄露 |
| 时间过滤 | G ≥ 20 | **MP ≥ 200** | 更精确的样本筛选 |
| 特征维度 | 8 维 | **12 维** | 更多有效信号 |
| 特征选择 | FG% | **eFG%** | 反映三分加权效率 |
| 3 超类性能 | Acc=0.758 | **Acc=0.827** | +6.8 个百分点 |

### 5.3 朴素贝叶斯独立性假设分析

朴素贝叶斯的核心假设是特征之间条件独立。通过相关性矩阵分析，本实验的 12 维特征中仅 AST_per36 与 AST_TOV 存在强相关（r=0.804），如果进一步优化可以考虑去掉其中一个。其余特征间相关性均在可接受范围内，且 per-36 化消除了原始数据中因上场时间引起的大量伪相关（如 PTS↔TRB 从 r=0.75 降至 r=-0.06）。

尽管如此，GaussianNB 在本实验中表现良好：5 分类 Accuracy=0.614（远超随机猜测的 20%），3 超类 Accuracy=0.827。即使在特征独立性假设不完全满足的情况下，朴素贝叶斯仍具有实用价值。

### 5.4 讨论

1. **per-36 归一化的有效性**：实验证明 per-36 化是比场均化更强的预处理策略。它不仅消除了上场时间带来的噪声，还意外地大幅改善了侧翼位置（SF）的分类效果。原因在于：SF 在场均数据中和 SG 的累计值接近，但在单位时间效率上存在差异——SF 的每 36 分钟篮板率更高，SG 的每 36 分钟得分效率更集中。

2. **5 位置与 3 超类的 trade-off**：3 超类虽然性能更优（Acc=0.827），但丢失了细粒度信息（如区分 C 和 PF）。实际应用中，应根据业务需求权衡：若只需区分"内线/侧翼/后卫"，3 超类足够；若需精确识别球员具体角色，则需接受 5 分类的较低准确率，或考虑使用非朴素贝叶斯的更复杂模型。

3. **朴素贝叶斯的局限性**：尽管预处理改进提升了性能，5 分类准确率仍仅有 61.4%。这反映了朴素贝叶斯算法本身的局限——条件独立假设在技术统计特征间不完全成立，且高斯分布假设可能不完全适合某些偏态分布的特征。

4. **未来改进方向**：
   - 引入身体测量数据（身高、臂展、体重），这些对位置判别可能有直接贡献；
   - 尝试半朴素贝叶斯（如 TAN、AODE）放松独立性假设；
   - 与其他算法（决策树、SVM、神经网络）进行对比实验，评估不同算法在位置预测任务上的优劣。

### 5.5 实验心得

通过本次实验，完整实践了从数据预处理到模型评估的朴素贝叶斯分类流程。per-36 分钟归一化的引入显著提升了分类性能，使我对特征工程中"如何消除噪声、提取纯净信号"有了更深刻的理解。TOT 去重的处理也展示了数据清洗中容易被忽视的细节——同一个体在数据集中多次出现可能导致训练集和测试集之间的信息泄露。

两组实验的对比进一步揭示了标签粒度对多分类任务性能的影响：同一算法、同一特征集，仅将标签从 5 类合并为 3 类，Accuracy 提升 21 个百分点。这提醒我们，当分类器表现不佳时，除调整算法和特征外，也应审慎评估类别标签的定义是否合理。
